### Passo 1: Verificar se o problema é do React.StrictMode

**Arquivo:** `src/main.tsx` ou `src/index.tsx`

**Procure por:**
```tsx

  <App />
</React.StrictMode>
```

**Teste temporário:**
```tsx
// Comente ou remova o StrictMode apenas para testar

  // <React.StrictMode>
    <App />
  // </React.StrictMode>
);
```

**Execute a aplicação e verifique:**
- Se as duplicações sumirem → O problema era o StrictMode (comportamento normal em dev)
- Se persistir → Continue com os próximos passos

#### 3.3 Modificar o useEffect de pré-carregamento

**Localizar:** O useEffect que começa na linha ~858

**SUBSTITUIR TODO O BLOCO:**
```tsx
useEffect(() => {
  // Coletar todos os documentos únicos de ambas as fontes
  const clientsFromVisits = selectedDateVisits.length > 0
    ? selectedDateVisits.map(v => v.clientDocument)
    : [];

  const clientsFromAvailable = availableClients.length > 0
    ? availableClients.map(c => c.document)
    : [];

  // Combinar e remover duplicatas
  const allUniqueClients = [...new Set([...clientsFromVisits, ...clientsFromAvailable])];

  // Filtrar apenas os que não estão no cache
  const missingClients = allUniqueClients.filter(doc => !clientDataCache.has(doc));

  if (missingClients.length > 0) {
    console.log(`🔄 Pré-carregando ${missingClients.length} clientes únicos...`);
    setIsLoadingClients(true);
    prefetchClientsData(missingClients).finally(() => {
      setIsLoadingClients(false);
    });

}, [selectedDateVisits, availableClients]);
```

**POR:**
```tsx
useEffect(() => {
  // ✅ Limpar timeout anterior
  if (prefetchTimeoutRef.current) {
    clearTimeout(prefetchTimeoutRef.current);
  }

  // ✅ Prevenir execução duplicada
  if (prefetchInProgressRef.current) {
    console.log("⏸️ Pré-carregamento já em andamento, pulando...");
    return;
  }

  // ✅ Debounce de 300ms para evitar chamadas rápidas
  prefetchTimeoutRef.current = setTimeout(() => {
    // Coletar todos os documentos únicos de ambas as fontes
    const clientsFromVisits = selectedDateVisits.length > 0
      ? selectedDateVisits.map(v => v.clientDocument)
      : [];

    const clientsFromAvailable = availableClients.length > 0
      ? availableClients.map(c => c.document)
      : [];

    // Combinar e remover duplicatas
    const allUniqueClients = [...new Set([...clientsFromVisits, ...clientsFromAvailable])];

    // Filtrar apenas os que não estão no cache
    const missingClients = allUniqueClients.filter(doc => !clientDataCache.has(doc));

    if (missingClients.length > 0) {
      console.log(`🔄 Pré-carregando ${missingClients.length} clientes únicos...`);
      
      // ✅ Bloquear novas execuções
      prefetchInProgressRef.current = true;
      setIsLoadingClients(true);
      
      prefetchClientsData(missingClients).finally(() => {
        setIsLoadingClients(false);
        // ✅ Liberar para novas execuções
        prefetchInProgressRef.current = false;
      });
    }
  }, 300); // Debounce de 300ms

  // ✅ Cleanup: limpar timeout ao desmontar ou quando dependências mudarem
  return () => {
    if (prefetchTimeoutRef.current) {
      clearTimeout(prefetchTimeoutRef.current);
    }
  };
}, [selectedDateVisits, availableClients, clientDataCache, prefetchClientsData]);
```

## 🔧 Solução de Problemas

### Problema: Ainda está duplicando

**Possíveis causas:**

1. **StrictMode ainda está ativo**
   - Verifique `src/main.tsx` ou `src/index.tsx`
   - Em desenvolvimento, o StrictMode causa duplicação intencional

2. **Múltiplos providers aninhados**
   - Verifique se `CollectionProvider` não está sendo renderizado 2x
   - Procure no `App.tsx` ou arquivo de rotas

3. **Hot Module Replacement (HMR)**
   - Durante desenvolvimento, HMR pode causar re-renderizações
   - Teste com build de produção: `npm run build && npm run preview`

### Problema: Erro de TypeScript

Se aparecer erro de tipo no `useRef`:
```tsx
// Adicione o tipo explícito:
const hasLoadedOnceRef = useRef<boolean>(false);
const prefetchInProgressRef = useRef<boolean>(false);
const prefetchTimeoutRef = useRef<NodeJS.Timeout | null>(null);
```

### Problema: Dados não carregam

Se os dados pararem de carregar:
```tsx
// Adicione um botão de reset no DevTools:
// No CollectionContext, exponha esta função:
const resetLoadingFlag = () => {
  hasLoadedOnceRef.current = false;
  setIsDataLoading(false);
};

// Use-a no console: window.resetCollectionLoading()
```

const fetchDataWithRetry = async (maxRetries = 3) => {
  for (let i = 0; i < maxRetries; i++) {
    try {
      await fetchData();
      break;
    } catch (error) {
